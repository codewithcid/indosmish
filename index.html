Architecture

How it works

Every step runs client-side. The message is tokenized, passed through a fine-tuned IndicBERT encoder compiled to ONNX, and classified — without a single network request after the one-time model download.

1

Input SMS

Raw code-mixed message, native or Roman script.

2

Tokenize

SentencePiece tokenizer, 200K-vocab, handles both scripts.

3

IndicBERT (ONNX)

12-layer Indic encoder runs on WebAssembly via transformers.js.

4

Softmax

Three-way probability over ham / spam / smishing.

5

Verdict

Label, confidence, and the persuasion signals detected.

The model

Fine-tuned IndicBERT, compiled for the edge

An Indic-specialized encoder, fine-tuned on a tri-class corpus and exported to ONNX so it can run at full precision inside any modern browser.

IndicBERT encoder

An ALBERT-style model pre-trained on 12 Indian languages — natively suited to Romanized and native-script Indic text.

ArchitectureALBERT · 12 layers
Vocabulary200K SentencePiece
Model size~135 MB (ONNX)

Tri-class fine-tuning
